## 一、数据元（Elements）字段对应关系

### 数据库表：rucp_element_biaozhun

| 数据库字段 | 含义 | DataFrame列名 | 向量库metadata字段 | 示例值 | 备注 |
|-----------|------|--------------|------------------|-------|------|
| code | 数据元标识符 | element_code | code | XM | ⭐ 用于返回 ename |
| chname | 中文名称 | cn_name | cn_name | 姓名 | ✅ |
| enname | 英文名称（拼音） | en_name | en_name | xing-ming | ✅ |

| length | 字段长度 | length | length | 30 | ✅ |
| valuetype | 值类型（分类） | classify | classify | 人物 | ✅ |
| interalcode | 内部标识符 | inner_code | element_code | DE00000002 | ⚠️ 作为向量库 ID |

### 向量库存储逻辑

```python
# build_index.py: build_elements()
ids = df.apply(lambda row: row.inner_code if row.inner_code else row.element_code, axis=1).tolist()
metas = [
    {
        "element_code": row.inner_code,  # ← 向量库 ID（内部标识符）
        "cn_name": row.cn_name,
        "en_name": row.en_name,          # ← 拼音
        "code": row.element_code,        # ⭐ 数据元标识符（如 XM）
        "type": row.type,
        "length": int(row.length),
        "classify": row.classify,
    }
]
```


## 二、限定词（Qualifiers）字段对应关系

```python
    }
]
```

### 字段对应关系图

```

================================================
chname → cn_name → cn_name           ✅ 中文名

code → identifier → code             ⭐ 与 identifier 相同
interalcode → inner_identifier → -   ⚠️ 未存储到向量库
```

## 三、关键问题说明

### ⚠️ 问题1：向量库缺少 `code` 字段

**原因：**
- 向量库在旧版本构建时，metadata 没有包含 `code` 字段
- 导致 `meta.get("code")` 返回 `None`

**解决方案：**
- 重新构建向量索引：`python scripts/build_index.py`

---

### ⚠️ 问题2：DataFrame 列名映射不一致

**数据库加载 vs 文件加载：**

#### 数据库加载（正确）
```python
# knowledge_loader.py: _load_from_db()
element_items.columns = ["element_code", "cn_name", "en_name", "type", "length", "classify", "inner_code"]
# ✅ element_code = 数据库的 code 字段
```

#### 文件加载（修复前）
```python
# knowledge_loader.py: _load_from_file() - 旧代码
element_items.columns = ["cn_name", "inner_code", "type", "length", "classify", "en_name"]
# ❌ inner_code 和 en_name 列名错误，导致 row.element_code 为空
```

#### 文件加载（修复后）
```python
# knowledge_loader.py: _load_from_file() - 新代码（已修复）
element_items.columns = ["cn_name", "element_code", "type", "length", "classify", "inner_code"]
# ✅ 列名正确，与数据库加载保持一致
```

## 📊 结构设计说明

### ✅ 为什么结构不同？

**element（数据元）** 和 **deteminer（限定词）** 的结构设计不同，这是合理的：

#### 1. element 的单层数组设计

```python
"cname": ["主记录ID", "姓名", "性别"]  # 单层数组
```

**原因：**
- 每个推荐返回多个候选数据元（Top-K 推荐）
- 数组中的每个元素代表一个候选数据元

#### 2. deteminer 的两层嵌套设计

```python
"cname": [["RUN"], ["采集地"]]     # 两层嵌套
"cname": [["RUN", "采集地"]]       # 一个限定词组合（多个限定词）
```

**原因：**
- 限定词可以组合使用（如 RUN + 采集地）
- 外层数组：多个限定词组合
- 内层数组：一个限定词组合中的多个限定词

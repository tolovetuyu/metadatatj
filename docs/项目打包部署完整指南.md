## 一、概述

```
┌─────────────────────────────────────────────────────────────┐
│                    Docker 容器                               │
├─────────────────────────────────────────────────────────────┤
```

### 2.1 开发环境要求

| 软件 | 版本 | 用途 |

| Python | >= 3.11 | 运行构建脚本 |

| 软件 | 版本 | 用途 |

| 资源 | 最低配置 | 推荐配置 |
|------|---------|---------|
| CPU | 2核 | 4核 |
| 内存 | 4GB | 8GB |
| 磁盘 | 10GB | 20GB |

### 2.4 安装 Docker

**CentOS/RHEL**
```bash
# 安装依赖
yum install -y yum-utils device-mapper-persistent-data lvm2

# 添加 Docker 源
yum-config-manager --add-repo https://download.docker.com/linux/centos/docker-ce.repo

# 安装 Docker
yum install -y docker-ce docker-ce-cli containerd.io

# 启动 Docker
systemctl start docker
systemctl enable docker

# 验证安装
docker --version
```

**Ubuntu/Debian**
```bash
# 更新包索引
apt-get update

# 安装依赖
apt-get install -y ca-certificates curl gnupg lsb-release

# 添加 Docker GPG 密钥
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | gpg --dearmor -o /usr/share/keyrings/docker-archive-keyring.gpg

# 添加 Docker 源
echo "deb [arch=$(dpkg --print-architecture) signed-by=/usr/share/keyrings/docker-archive-keyring.gpg] https://download.docker.com/linux/ubuntu $(lsb_release -cs) stable" | tee /etc/apt/sources.list.d/docker.list > /dev/null

# 安装 Docker
apt-get update
apt-get install -y docker-ce docker-ce-cli containerd.io

# 启动 Docker
systemctl start docker
systemctl enable docker

# 验证安装
docker --version
```

**Windows**
```powershell
# 下载并安装 Docker Desktop
# https://www.docker.com/products/docker-desktop

# 或使用 Chocolatey
choco install docker-desktop
```

## 三、项目构建

### 3.1 获取项目代码

**方式一：Git 克隆**
```bash
git clone <repository-url> metadatatj
cd metadatatj
```

**方式二：直接复制**
```bash
# 复制项目目录到目标位置
cp -r /path/to/metadatatj /opt/metadatatj
cd /opt/metadatatj
```

### 3.2 项目目录结构

```
metadatatj/
├── docker/                    # Docker 配置
│   ├── Dockerfile            # 镜像定义
│   ├── docker-compose.yml    # 编排配置
│   └── entrypoint.sh         # 启动脚本
├── scripts/                   # 脚本
│   └── build_index.py        # 构建向量索引
├── src/                       # 源代码
│   ├── app_recommend.py
│   ├── config.py
│   └── ...
├── data/                      # 数据目录
│   └── chroma/               # 向量库数据
├── logs/                      # 日志目录
├── .env                       # 环境配置
├── .env.example              # 配置模板
└── requirements.txt          # Python 依赖
```

### 3.3 配置环境变量

```bash
# 复制配置模板
cp .env.example .env

# 编辑配置
vim .env
```

**关键配置项**：

```bash

HISTORY_DB_USER=root
HISTORY_DB_PASSWORD=your_password

# ========== Embedding 配置 ==========

EMBEDDING_API_KEY=sk-your-api-key
EMBEDDING_MODEL=text-embedding-v3

# ========== 历史推荐配置 ==========
```

### 3.4 构建向量索引（可选）

如果使用文件模式或有预置的知识库数据，需要预先构建向量索引：

```bash
# 安装依赖
pip install -r requirements.txt

# 构建向量索引
python scripts/build_index.py

# 验证索引
ls -la data/chroma/
# 应看到 chroma.sqlite3 文件
```

**注意**：数据库模式下，向量索引会在首次启动时自动构建。

## 五、镜像分发

### 5.1 在线分发（Docker Hub/私有仓库）

**推送到 Docker Hub**：
```bash
# 登录 Docker Hub
docker login

# 推送镜像
docker push metadatatj:latest
docker push metadatatj:v1.0.0
```

**推送到私有仓库**：
```bash
# 标记镜像
docker tag metadatatj:latest registry.example.com/metadatatj:latest

# 推送到私有仓库
docker push registry.example.com/metadatatj:latest
```

### 5.2 离线分发（导出/导入）

**导出镜像**：
```bash
# 导出为 tar 文件
docker save metadatatj:latest -o metadatatj-latest.tar

# 或使用 gzip 压缩（推荐）
docker save metadatatj:latest | gzip > metadatatj-latest.tar.gz

# 查看文件大小
ls -lh metadatatj-latest.tar.gz
# 约 180MB（压缩后）
```

**传输镜像**：
```bash
# 方式一：SCP 传输
scp metadatatj-latest.tar.gz user@server:/tmp/

# 方式二：U盘/移动硬盘
cp metadatatj-latest.tar.gz /media/usb/

# 方式三：HTTP 下载（放在 Web 服务器）
mv metadatatj-latest.tar.gz /var/www/html/downloads/
```

**导入镜像**（在生产环境）：
```bash
# 导入镜像
docker load -i metadatatj-latest.tar.gz

# 或解压后导入
gunzip -c metadatatj-latest.tar.gz | docker load

# 验证导入
docker images metadatatj
```

### 5.3 分发清单

创建分发包：

```bash
# 创建分发目录
mkdir -p dist/metadatatj-v1.0.0

# 复制文件
cp metadatatj-latest.tar.gz dist/metadatatj-v1.0.0/
cp .env.example dist/metadatatj-v1.0.0/
cp docker/docker-compose.yml dist/metadatatj-v1.0.0/
cp -r knowledge dist/metadatatj-v1.0.0/

# 创建部署脚本
cat > dist/metadatatj-v1.0.0/deploy.sh << 'EOF'
#!/bin/bash
# 部署脚本

# 1. 导入镜像
echo "导入 Docker 镜像..."
docker load -i metadatatj-latest.tar.gz

# 2. 创建目录
echo "创建数据目录..."
mkdir -p /opt/metadatatj/{data,logs}

# 3. 配置环境变量
echo "请编辑 .env 文件配置数据库连接"
cp .env.example .env

echo "部署完成！"
echo "下一步："
echo "  1. 编辑 .env 配置数据库连接"
echo "  2. 运行: docker-compose up -d"
EOF

chmod +x dist/metadatatj-v1.0.0/deploy.sh

# 打包分发
cd dist
tar -czf metadatatj-v1.0.0.tar.gz metadatatj-v1.0.0/
```

### 6.3 配置环境变量

创建 `.env` 文件：

```bash
cat > /opt/metadatatj/.env << 'EOF'
# ========== 服务端口 ==========
RECOMMEND_PORT=6058
TABLEMAP_PORT=6059
ADMIN_PORT=6060

# ========== 数据源配置 ==========
DATA_SOURCE=db

# 主数据库
DB_HOST=192.168.1.100
DB_PORT=3306
DB_USER=root
DB_PASSWORD=your_password
DB_NAME=metadata

# 历史数据库
HISTORY_DB_HOST=192.168.1.100
HISTORY_DB_PORT=3306
HISTORY_DB_USER=root
HISTORY_DB_PASSWORD=your_password
HISTORY_DB_NAME=metadata_history

# ========== LLM 配置 ==========
LLM_API_BASE=https://dashscope.aliyuncs.com/compatible-mode/v1
LLM_API_KEY=sk-your-api-key
LLM_MODEL=qwen-plus
LLM_TIMEOUT=60

# ========== Embedding 配置 ==========
EMBEDDING_API_BASE=https://dashscope.aliyuncs.com/compatible-mode/v1
EMBEDDING_API_KEY=sk-your-api-key
EMBEDDING_MODEL=text-embedding-v3
EMBEDDING_BATCH_SIZE=64

# ========== 推荐配置 ==========
RECALL_TOP_K=30
RERANK_TOP_K=5
LLM_DECOMPOSE_ENABLED=true
LLM_RERANK_ENABLED=true

# ========== 历史推荐配置 ==========
HISTORY_RECOMMEND_ENABLED=true
HISTORY_SYNC_INTERVAL_HOURS=24
EOF
```

### 6.4 Docker Compose 配置

创建 `docker-compose.yml`：

```yaml
version: "3.8"

services:
  metadatatj:
    image: metadatatj:latest
    container_name: metadatatj
    restart: unless-stopped
    ports:
      - "${RECOMMEND_PORT:-6058}:6058"
      - "${TABLEMAP_PORT:-6059}:6059"
      - "${ADMIN_PORT:-6060}:6060"
    volumes:
      - ./data:/app/data
      - ./logs:/app/logs
    env_file:
      - .env
    environment:
      # 容器内访问宿主机数据库
      - DB_HOST=${DB_HOST:-host.docker.internal}
      - HISTORY_DB_HOST=${HISTORY_DB_HOST:-host.docker.internal}
    deploy:
      resources:
        limits:
          memory: 8G
          cpus: '4'
    logging:
      driver: json-file
      options:
        max-size: "100m"
        max-file: "10"
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:6058/autoexport/api/recommend", "-X", "POST", "-H", "Content-Type: application/json", "-d", "{\"fieldsInfo\":[]}"]
      interval: 30s
      timeout: 10s
      retries: 3
      start_period: 60s
```

## 七、启动服务

### 7.1 使用 Docker Compose 启动（推荐）

```bash
cd /opt/metadatatj

# 启动服务
docker-compose up -d

# 查看日志
docker-compose logs -f

# 查看状态
docker-compose ps
```

### 7.2 使用 Docker 命令启动

```bash
docker run -d \
    --name metadatatj \
    --restart unless-stopped \
    -p 6058:6058 \
    -p 6059:6059 \
    -p 6060:6060 \
    -v /opt/metadatatj/data:/app/data \
    -v /opt/metadatatj/logs:/app/logs \
    --env-file /opt/metadatatj/.env \
    -e DB_HOST=host.docker.internal \
    --memory 8g \
    --cpus 4 \
    metadatatj:latest all
```

### 7.3 启动单个服务

```bash
# 仅启动推荐服务
docker run -d \
    --name metadatatj-recommend \
    -p 6058:6058 \
    -v /opt/metadatatj/data:/app/data \
    --env-file /opt/metadatatj/.env \
    metadatatj:latest recommend

# 仅启动表映射服务
docker run -d \
    --name metadatatj-tablemap \
    -p 6059:6059 \
    -v /opt/metadatatj/data:/app/data \
    --env-file /opt/metadatatj/.env \
    metadatatj:latest tablemap

# 仅启动管理后台
docker run -d \
    --name metadatatj-admin \
    -p 6060:6060 \
    -v /opt/metadatatj/data:/app/data \
    --env-file /opt/metadatatj/.env \
    metadatatj:latest admin
```

### 7.4 启动参数说明

| 参数 | 说明 |
|------|------|
| `all` | 启动所有服务（默认） |
| `recommend` | 仅启动推荐服务 |
| `tablemap` | 仅启动表映射服务 |
| `admin` | 仅启动管理后台 |
| `build-index` | 仅构建向量索引 |

## 十、常见问题

### 10.1 容器无法启动

**问题**：容器启动后立即退出

**排查步骤**：
```bash
# 1. 查看日志
docker logs metadatatj

# 2. 检查配置
cat /opt/metadatatj/.env

# 3. 检查数据库连接
docker exec metadatatj ping DB_HOST

# 4. 检查端口占用
netstat -tlnp | grep -E '6058|6059|6060'
```

**常见原因**：
- 数据库连接失败 → 检查 DB_HOST/DB_PASSWORD
- LLM API 配置错误 → 检查 LLM_API_KEY
- 端口被占用 → 修改端口配置
- 内存不足 → 增加内存限制

### 10.2 向量索引构建失败

**问题**：启动时报错 "向量索引不存在"

**解决方法**：
```bash
# 方式一：进入容器构建
docker exec -it metadatatj python scripts/build_index.py

# 方式二：启动时自动构建
# entrypoint.sh 会自动检测并构建
```

### 10.3 数据库连接失败

**问题**：DATA_SOURCE=db 时连接失败

**解决方法**：
```bash
# 检查网络连通性
docker exec metadatatj ping DB_HOST

# 使用 host.docker.internal 访问宿主机
docker run -e DB_HOST=host.docker.internal ...

# 或使用宿主机 IP
docker run -e DB_HOST=192.168.1.100 ...

# 检查防火墙
firewall-cmd --list-ports
```

### 10.4 内存不足

**问题**：容器因内存不足被 OOM Kill

**解决方法**：
```bash
# 增加内存限制
docker update --memory 8g --memory-swap 8g metadatatj

# 或启动时指定
docker run --memory 8g --memory-swap 8g ...

# 查看内存使用
docker stats metadatatj
```

### 10.5 LLM API 调用失败

**问题**：推荐返回空结果或报错

**排查步骤**：
```bash
# 1. 检查 API Key
echo $LLM_API_KEY

# 2. 测试 API 连接
curl -X POST $LLM_API_BASE/chat/completions \
    -H "Authorization: Bearer $LLM_API_KEY" \
    -H "Content-Type: application/json" \
    -d '{"model": "qwen-plus", "messages": [{"role": "user", "content": "test"}]}'

# 3. 查看日志
docker logs metadatatj | grep -i error
```

### 10.6 Windows 路径问题

**问题**：Windows 下挂载路径错误

**解决方法**：
```powershell
# 使用绝对路径
docker run -v C:/data/metadatatj/data:/app/data ...

# 或使用 ${PWD}
docker run -v ${PWD}/data:/app/data ...

# PowerShell 中
docker run -v "$PWD/data:/app/data" ...
```

### 10.7 离线部署问题

**问题**：导入镜像时报错

**解决方法**：
```bash
# 检查文件完整性
md5sum metadatatj-latest.tar.gz

# 解压后导入
gunzip -c metadatatj-latest.tar.gz | docker load

# 检查磁盘空间
df -h
```

## 附录A：完整部署脚本

```bash
#!/bin/bash
# deploy.sh - MetadataTJ 部署脚本

set -e

# 配置
IMAGE_FILE="metadatatj-latest.tar.gz"
INSTALL_DIR="/opt/metadatatj"
IMAGE_NAME="metadatatj:latest"

echo "========================================"
echo "MetadataTJ 部署脚本"
echo "========================================"

# 1. 检查 Docker
echo "[1/6] 检查 Docker..."
if ! command -v docker &> /dev/null; then
    echo "错误: Docker 未安装"
    exit 1
fi
echo "Docker 版本: $(docker --version)"

# 2. 导入镜像
echo "[2/6] 导入 Docker 镜像..."
if [ -f "$IMAGE_FILE" ]; then
    docker load -i "$IMAGE_FILE"
    echo "镜像导入成功"
else
    echo "错误: 镜像文件不存在: $IMAGE_FILE"
    exit 1
fi

# 3. 创建目录
echo "[3/6] 创建目录..."
mkdir -p "$INSTALL_DIR"/{data,logs}
chmod -R 755 "$INSTALL_DIR"
echo "目录创建成功"

# 4. 配置环境变量
echo "[4/6] 配置环境变量..."
if [ ! -f "$INSTALL_DIR/.env" ]; then
    cp .env.example "$INSTALL_DIR/.env"
    echo "已创建 .env 文件，请编辑配置"
else
    echo ".env 文件已存在"
fi

# 5. 创建 docker-compose.yml
echo "[5/6] 创建 docker-compose.yml..."
cat > "$INSTALL_DIR/docker-compose.yml" << 'EOF'
version: "3.8"
services:
  metadatatj:
    image: metadatatj:latest
    container_name: metadatatj
    restart: unless-stopped
    ports:
      - "${RECOMMEND_PORT:-6058}:6058"
      - "${TABLEMAP_PORT:-6059}:6059"
      - "${ADMIN_PORT:-6060}:6060"
    volumes:
      - ./data:/app/data
      - ./logs:/app/logs
    env_file:
      - .env
    deploy:
      resources:
        limits:
          memory: 8G
          cpus: '4'
EOF
echo "docker-compose.yml 创建成功"

# 6. 完成
echo "[6/6] 部署完成！"
echo ""
echo "下一步操作："
echo "  1. 编辑配置文件: vim $INSTALL_DIR/.env"
echo "  2. 启动服务: cd $INSTALL_DIR && docker-compose up -d"
echo "  3. 查看日志: docker-compose logs -f"
echo "  4. 测试接口: curl http://localhost:6058/autoexport/api/recommend"
```

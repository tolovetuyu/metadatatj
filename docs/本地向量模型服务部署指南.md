# 本地向量模型服务部署指南

## 概述

本方案使用 FastEmbed + FastAPI 搭建本地向量模型服务，完全兼容 OpenAI Embedding API 格式，适用于内网无外网连接的环境。

## 模型选择

| 模型 | 维度 | 大小 | 性能 | 推荐场景 |
|------|------|------|------|----------|
| BAAI/bge-small-zh-v1.5 | 512 | ~100MB | 快速 | 资源受限环境 |
| BAAI/bge-base-zh-v1.5 | 768 | ~400MB | 平衡 | **推荐默认使用** |
| BAAI/bge-large-zh-v1.5 | 1024 | ~1.2GB | 高性能 | 高精度需求 |

## 方式一：单独部署向量服务

### 1. 构建向量服务镜像

```bash
cd embedding_service
docker build -t embedding-service:latest .
```

### 2. 启动向量服务

```bash
docker run -d \
  --name embedding-service \
  --restart unless-stopped \
  -p 8000:8000 \
  -v embedding-cache:/root/.cache \
  embedding-service:latest
```

### 3. 验证服务

```bash
# 健康检查
curl http://localhost:8000/

# 测试向量计算
curl -X POST http://localhost:8000/v1/embeddings \
  -H 'Content-Type: application/json' \
  -d '{"model": "BAAI/bge-base-zh-v1.5", "input": "测试文本"}'
```

### 4. 配置主服务使用本地向量服务

修改 `.env` 文件：

```bash
EMBEDDING_API_BASE=http://localhost:8000/v1
EMBEDDING_API_KEY=local
EMBEDDING_MODEL=BAAI/bge-base-zh-v1.5
EMBEDDING_BATCH_SIZE=10
```

## 方式三：内网离线部署

### 1. 在有外网的机器上准备镜像

```bash
# 构建向量服务镜像
cd embedding_service
docker build -t embedding-service:latest .

# 导出镜像
docker save embedding-service:latest -o embedding-service.tar

# 同时导出主服务镜像
docker save metadatatj:latest -o metadatatj.tar
```

### 2. 上传到内网服务器

```bash
# 上传两个 tar 文件到内网服务器
scp embedding-service.tar metadatatj.tar user@internal-server:/opt/
```

### 3. 在内网服务器加载镜像

```bash
# 加载向量服务镜像
docker load -i embedding-service.tar

# 加载主服务镜像
docker load -i metadatatj.tar
```

### 4. 启动服务

```bash
# 启动向量服务
docker run -d \
  --name embedding-service \
  --restart unless-stopped \
  -p 8000:8000 \
  -v embedding-cache:/root/.cache \
  embedding-service:latest

# 等待向量服务启动完成（约 60-120 秒）
docker logs -f embedding-service

# 启动主服务（配置使用本地向量服务）
docker run -d \
  --name metadatatj \
  --restart unless-stopped \
  -p 7058:7058 \
  -p 7059:7059 \
  -p 7060:7060 \
  -v /opt/metadataTJ/data:/app/data \
  -v /opt/metadataTJ/logs:/app/logs \
  -v /opt/metadataTJ/knowledge:/app/knowledge \
  -e EMBEDDING_API_BASE=http://localhost:8000/v1 \
  -e EMBEDDING_API_KEY=local \
  -e EMBEDDING_MODEL=BAAI/bge-base-zh-v1.5 \
  -e EMBEDDING_BATCH_SIZE=10 \
  --env-file /opt/metadataTJ/.env \
  --link embedding-service:embedding-service \
  metadatatj:latest
```

## 性能优化建议

### 1. 预下载模型

在有外网的机器上预下载模型，然后复制到内网：

```bash
# 在有外网的机器上
docker run -it --rm -v embedding-cache:/root/.cache embedding-service:latest \
  python -c "from fastembed import TextEmbedding; TextEmbedding('BAAI/bge-base-zh-v1.5')"

# 导出缓存
docker run -it --rm -v embedding-cache:/cache -v $(pwd):/output embedding-service:latest \
  cp -r /cache /output/embedding-cache

# 上传到内网服务器并加载
```

### 2. 资源配置

| 配置项 | 推荐值 | 说明 |
|--------|--------|------|
| 内存 | 2GB+ | 向量模型需要足够内存 |
| CPU | 2核+ | 并发计算需要 |
| 缓存卷 | 持久化 | 避免每次重启重新加载模型 |

## 常见问题

### Q1: 模型下载慢怎么办？

**解决方案**：在有外网的机器上预下载模型，然后复制缓存目录到内网。

### Q2: 内存不足怎么办？

**解决方案**：使用更小的模型 `BAAI/bge-small-zh-v1.5`，或增加容器内存限制。

### Q3: 向量维度不一致怎么办？

**解决方案**：确保主服务和向量服务使用相同的模型名称。

## API 接口说明

### 健康检查

```
GET http://localhost:8000/
```

### 列出可用模型

```
GET http://localhost:8000/v1/models
```

### 计算向量

```
POST http://localhost:8000/v1/embeddings
Content-Type: application/json

{
  "model": "BAAI/bge-base-zh-v1.5",
  "input": "测试文本"  // 或 ["文本1", "文本2"]
}
```

**响应格式**（兼容 OpenAI）：

```json
{
  "object": "list",
  "data": [
    {
      "object": "embedding",
      "index": 0,
      "embedding": [0.1, 0.2, ...]
    }
  ],
  "model": "BAAI/bge-base-zh-v1.5",
  "usage": {
    "prompt_tokens": 10,
    "total_tokens": 10
  }
}
```

# MetadataTJ 推荐逻辑完整分析

## 一、推荐流程总览

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                          推荐流程（三阶段流水线）                              │
└─────────────────────────────────────────────────────────────────────────────┘

输入: cname (中文名), ename (英文名)
    │
    ├─→ 1. 字段语义分解 (LLM)
    │       └─→ decompose_field()
    │           - 提取核心数据元
    │           - 提取限定词
    │           - 生成语义提示
    │
    ├─→ 2. 构建检索查询
    │       └─→ build_retrieval_queries()
    │           - 数据元查询列表
    │           - 限定词查询列表
    │
    ├─→ 3. 照片特殊规则
    │       └─→ _apply_photo_rule()
    │           - 照片 → 电子文件存放路径
    │
    ├─→ 4. 向量召回 (ChromaDB)
    │       └─→ store.search_elements()
    │           - 召回 TopK 候选数据元
    │           - 基于语义相似度
    │
    ├─→ 5. 历史推荐合并 ⭐
    │       └─→ _history.merge_with_candidates()
    │           - 查询历史对标记录
    │           - 按匹配次数排序
    │           - 历史结果优先插入候选列表
    │
    ├─→ 6. LLM 精排
    │       └─→ rerank_elements()
    │           - 从候选中选出最佳 Top5
    │           - 基于语义相关性打分
    │
    ├─→ 7. 构建返回结果
    │       ├─→ _resolve_element_row()  # 构建数据元行
    │       ├─→ _build_gz()             # 格式化函数
    │       └─→ GYH_MAP                 # 统一化函数
    │
    ├─→ 8. 限定词推荐（如有）
    │       ├─→ store.search_qualifiers()  # 向量召回
    │       └─→ rerank_qualifiers()       # LLM精排
    │
    └─→ 输出: 推荐结果
```

## 三、推荐流程详解

### 3.1 字段语义分解

**函数**: `decompose_field(cname, ename, llm)`

**作用**: 使用 LLM 分析字段语义，提取核心数据元和限定词。

**输入示例**:
```
cname = "父亲姓名"
ename = "FQXM"
```

**输出**:
```python
FieldDecomposition(
    core_element_hint="姓名",      # 核心数据元
    raw_determiners=["父亲"],      # 限定词
    semantic_hints=["亲属关系"]    # 语义提示
)
```

**特殊处理**:
```python
# 证照代码规则
if core_element_hint in ZK_CODES:
    decomp = decompose_field(cname + "代码", ename, llm)

# 扩展字段规则
if extend_label and not core_element_hint.endswith("代码"):
    raw_determiners.append(core_element_hint)
    core_element_hint = "信息代码"
```

---

### 3.2 照片特殊规则

**函数**: `_apply_photo_rule(top_cn, element_query)`

**作用**: 照片字段强制映射到电子文件存放路径。

```python
def _apply_photo_rule(self, top_cn: str, element_query: str) -> str:
    if top_cn == "照片":
        return "电子文件存放路径"
    return element_query
```

---

### 3.3 向量召回

**函数**: `store.search_elements(queries, top_k)`

**作用**: 从 ChromaDB 向量库中召回相似的数据元。

**流程**:
```
queries = ["姓名", "人员姓名", "姓名代码"]
    ↓
ChromaDB 向量检索
    ↓
candidates = [
    {"element_code": "DE0001", "cn_name": "姓名", "score": 0.95},
    {"element_code": "DE0002", "cn_name": "人员姓名", "score": 0.92},
    ...
]
```

---

### 3.4 历史推荐合并 ⭐

**函数**: `_history.merge_with_candidates(cname, ename, candidates, top_k)`

**作用**: 将历史对标记录合并到候选列表，历史结果优先。

**数据来源**: `rucp_element_mapping_history` 表

**流程**:
```
1. 查询历史对标记录
   SELECT target_element_code, match_count
   FROM rucp_history_recommend_stat
   WHERE source_cname = '姓名'
   ORDER BY match_count DESC

2. 按匹配次数排序
   history_results = [
       {"element_code": "DE0001", "match_count": 150},
       {"element_code": "DE0003", "match_count": 80},
   ]

3. 合并到候选列表（历史优先）
   candidates = history_results + vector_results
```

**优先级**:
```
历史推荐结果 > 向量召回结果
```

---

### 3.5 LLM 精排

**函数**: `rerank_elements(cname, ename, core_hint, candidates, top_k, llm)`

**作用**: 使用 LLM 从候选中选出最佳 TopK。

**流程**:
```
candidates = [30个候选]
    ↓
LLM Prompt: "从以下数据元中选出与'姓名'最相关的5个..."
    ↓
rankings = [
    {"element_code": "DE0001", "score": 0.98},
    {"element_code": "DE0002", "score": 0.95},
    ...
]
```

---

**数据元结果**:

```python
{
    "element": {

    }

```

### 3.7 限定词推荐

**触发条件**: 字段分解结果包含限定词

**流程**:
```
det_queries = ["父亲", "母亲"]
    ↓
for det_q in det_queries:
    ├─→ store.search_qualifiers([det_q], top_k)  # 向量召回
    └─→ rerank_qualifiers(det_q, candidates, top_k, llm)  # LLM精排
```

**结果**:
```python
{
    "deteminer": {
        "cname": [["父亲"], ["母亲"]],
        "ename": [["FQ"], ["MQ"]],
        "label": [[0], [0]],
        "score": [[0.95], [0.92]]
    }
}
```

```
│     - 来源: ChromaDB 向量检索                                │
```

## 十二、错误处理

```python
try:
    # 字段分解
    decomp = decompose_field(cname, ename, self.llm)
except Exception:
    # 降级：使用原始字段名
    decomp = FieldDecomposition(core_element_hint=cname)

try:
    # 向量召回
    candidates = self.store.search_elements(queries, top_k)
except Exception:
    # 降级：返回空候选
    candidates = []
```

## 十三、总结

MetadataTJ 推荐系统采用**三阶段流水线**架构：

1. **语义分解**：LLM 理解字段语义
2. **向量召回**：ChromaDB 召回候选
3. **精排输出**：LLM 选出最佳结果

**核心优势**：
- 历史推荐优先，充分利用人工经验
- LLM 语义理解，处理复杂字段
- 向量检索高效，支持大规模数据
- 数据库数据源，便于维护更新

为什么导入时必须向量化？

  • 检索原理：推荐服务用「查询文本的向量」与「库内向量」做余弦相似度，没有向量就无法召回。
  • 模型一致：入库与查询须使用同一 Embedding 模型（当前：{{ embedding_model }}），否则相似度无意义。
  • 导入流程：Excel 解析 → 生成检索文本 → 批量调用 Embedding API → 向量 + 元数据写入 Chroma。
  • 更新策略：修改 Excel 后重新导入即可；相同 ID 会覆盖旧向量（upsert 模式）。
